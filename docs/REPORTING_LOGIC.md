# หลักการคำนวณค่าสถิติในรายงาน (Reporting Logic Calculation)

เอกสารนี้อธิบายวิธีการคำนวณตัวเลขและค่าสถิติต่างๆ ที่แสดงในหน้า Dashboard และหน้า Reports เพื่อความเข้าใจที่ตรงกัน

## 1. Ticket ทั้งหมด (Total Tickets)
* **ความหมาย:** จำนวน Ticket ที่ถูกสร้างขึ้นในช่วงเวลาที่เลือก (Date Range)
* **เกณฑ์การนับ:** นับจากวันที่ `Created Date`

## 2. อัตราการแก้ไข Ticket (Resolution Rate)
* **ความหมาย:** สัดส่วนของ Ticket ที่ได้รับการแก้ไขสำเร็จ เทียบกับจำนวน Ticket ทั้งหมดที่เข้ามาในช่วงเวลานั้น
* **สูตรคำนวณ:** `(จำนวน Ticket ที่แก้ไขเสร็จ / Ticket ทั้งหมดในช่วงเวลา) * 100`
* **เกณฑ์การนับ Ticket ที่แก้ไขเสร็จ (Effective Resolved):**
  1. Ticket ที่มีสถานะปัจจุบันเป็น **Resolved**
  2. Ticket ที่มีสถานะปัจจุบันเป็น **Closed** **และ** มีวันที่แก้ไขเสร็จ (`ResolvedAt`) ระบุไว้ (เพื่อแยกออกจาก Ticket ที่ถูกปิดแบบ Cancel หรือปิดโดยไม่ได้แก้)

## 3. เวลาตอบสนองเฉลี่ย (Average Response Time)
* **ความหมาย:** ระยะเวลาโดยเฉลี่ยตั้งแต่เริ่มเปิด Ticket จนถึงเวลาที่เริ่มปฏิบัติงาน (เปลี่ยนสถานะเป็น In Progress)
* **กลุ่มข้อมูลที่ใช้ (Cohort):** Ticket ที่ถูก**สร้างขึ้นในช่วงเวลาที่เลือก** และมีการเริ่มงานแล้ว (`InProgressAt` มีค่า)
* **สูตรคำนวณ:** `ผลรวมของ (InProgressAt - CreatedAt) / จำนวน Ticket ที่เริ่มงานแล้ว`
* **หน่วย:** นาที หรือ ชั่วโมง

## 4. เวลาแก้ไขปัญหาเฉลี่ย (Average Resolution Time)
* **ความหมาย:** ระยะเวลาโดยเฉลี่ยตั้งแต่เริ่มเปิด Ticket จนถึงเวลาที่แก้ไขปัญหาเสร็จสิ้น
* **กลุ่มข้อมูลที่ใช้ (Performance):** Ticket ที่ **จบงาน (Resolved หรือ Closed) ในช่วงเวลาที่เลือก** (ไม่คำนึงว่าสร้างเมื่อไหร่) เพื่อสะท้อนผลงานการปิดงานในรอบเดือนนั้นๆ
* **สูตรคำนวณ:** `ผลรวมของ ((ResolvedAt หรือ ClosedAt) - CreatedAt) / จำนวน Ticket ที่จบงาน`
* **หน่วย:** นาที หรือ ชั่วโมง

## 5. การปฏิบัติตาม SLA (SLA Compliance)
* **ความหมาย:** เปอร์เซ็นต์ของ Ticket ที่สามารถแก้ไขปัญหาได้เสร็จสิ้นภายในเวลาที่กำหนด (SLA Deadline)
* **กลุ่มข้อมูลที่ใช้:** Ticket ที่ **จบงาน (Resolved หรือ Closed) ในช่วงเวลาที่เลือก** และมีกำหนดการ SLA (`SLA Deadline`)
* **เกณฑ์การตัดสิน:**
  * **ผ่าน (Met SLA):** เวลาที่จบงาน (`ResolvedAt` หรือ `ClosedAt`) **น้อยกว่าพรือเท่ากับ** `SLA Deadline`
  * **ไม่ผ่าน (Missed SLA):** เวลาที่จบงาน **มากกว่า** `SLA Deadline`
* **สูตรคำนวณ:** `(จำนวน Ticket ที่ผ่าน SLA / จำนวน Ticket ที่มี SLA ทั้งหมดที่จบงานในช่วงเวลา) * 100`

## 6. สรุปความแตกต่างของชุดข้อมูล (Data Scope)

| ค่าสถิติ | ใช้ข้อมูล Ticket แบบไหน? | เพื่อวัดอะไร? |
| :--- | :--- | :--- |
| **Total Tickets** | สร้างในช่วงเวลาที่เลือก (Created) | ปริมาณงานขาเข้า (Volume) |
| **Resolution Rate** | สร้างในช่วงเวลาที่เลือก (Created) | ประสิทธิภาพการเคลียร์งานที่เข้ามา (Throughput Rate) |
| **Avg Response Time** | สร้างในช่วงเวลาที่เลือก (Created) | ความรวดเร็วในการตอบรับงานปัจจุบัน |
| **Avg Resolution Time** | จบงานในช่วงเวลาที่เลือก (Resolved/Closed) | ความเร็วในการแก้ปัญหาของทีมงานในช่วงนี้ |
| **SLA Compliance** | จบงานในช่วงเวลาที่เลือก (Resolved/Closed) | คุณภาพการให้บริการของงานที่ส่งมอบแล้ว |
